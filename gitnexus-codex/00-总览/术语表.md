# 术语表

| 术语               | 含义                                | 分享时怎么讲                    |
| ---------------- | --------------------------------- | ------------------------- |
| Repository       | 被索引的代码仓库                          | 原材料                       |
| Symbol           | 函数、类、方法、接口等代码元素                   | 图谱节点                      |
| Relation         | CALLS、IMPORTS、DEFINES、EXTENDS 等关系 | 图谱边                       |
| KnowledgeGraph   | 内存中的代码知识图谱                        | 索引阶段的主数据结构                |
| LadybugDB        | 本地图数据库文件                          | 图谱落盘后的查询后端                |
| Pipeline         | analyze 阶段的有向无环执行管线               | 把源码一步步加工成图谱               |
| Tree-sitter      | 多语言 AST 解析器                       | 从文本进入结构化世界的入口             |
| Scope Resolution | 新一代引用解析管线                         | 先建全局作用域索引，再解析引用           |
| MRO              | Method Resolution Order           | 处理继承、重写、实现关系              |
| Community        | Leiden 聚类得到的功能模块                  | 自动发现的代码功能区                |
| Process          | 从路由或工具出发追踪出的执行流                   | 适合 Agent 理解业务流程           |
| MCP              | Model Context Protocol            | Agent 调用 GitNexus 的标准工具协议 |
| Skill            | 面向任务的工作流提示文件                      | 让 Agent 知道“遇到某类任务该怎么做”    |
| AGENTS.md        | 项目级 Agent 规则                      | MUST、NEVER、ALWAYS 级约束     |
| Contract         | 跨仓库服务契约                           | HTTP、gRPC、Topic 等边界       |
| ContractLink     | 契约匹配后的跨仓库连接                       | Group 跨仓库影响分析的桥           |

## 容易混淆

GitNexus 不是只做 Embedding。Embedding 是搜索排序的一部分，核心能力仍然来自静态分析得到的确定性关系。

LadybugDB 是本地持久化图存储，不是远程 SaaS。默认模型是本地优先，数据留在项目目录和用户目录下。

Skill 不是代码索引结果本身，而是把索引结果转化成 Agent 行为约束的一层提示词工程。
