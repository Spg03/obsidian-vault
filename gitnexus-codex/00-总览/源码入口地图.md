# 源码入口地图

这篇是技术分享时最有用的“源码导航”。它不是完整文件列表，而是告诉你每个能力该从哪里读起。

## CLI 与 analyze

| 主题 | 源码入口 | 读什么 |
|---|---|---|
| analyze 命令参数 | `gitnexus/src/cli/analyze.ts` | Commander 参数、embeddings、skills、index-only 等选项 |
| analyze 总编排 | `gitnexus/src/core/run-analyze.ts` | `runFullAnalysis`：早期返回、Pipeline、LadybugDB、FTS、Embedding、AI context |
| repo 元信息 | `gitnexus/src/storage/repo-manager.ts` | `.gitnexus` 路径、meta、registry、gitignore |
| parse cache | `gitnexus/src/storage/parse-cache.ts` | 内容寻址缓存，避免重复解析 |

## Pipeline

| 主题 | 源码入口 | 读什么 |
|---|---|---|
| Phase 列表 | `gitnexus/src/core/ingestion/pipeline.ts` | `buildPhaseList` 中的 13 个阶段 |
| DAG runner | `gitnexus/src/core/ingestion/pipeline-phases/runner.ts` | Kahn 拓扑排序、依赖输出隔离、错误包装 |
| Phase 类型 | `gitnexus/src/core/ingestion/pipeline-phases/types.ts` | `PipelinePhase`、`PipelineContext`、`PhaseResult` |
| parse 阶段 | `gitnexus/src/core/ingestion/pipeline-phases/parse.ts` | worker pool、parse implementation、缓存 |
| scope resolution | `gitnexus/src/core/ingestion/scope-resolution/pipeline/phase.ts` | 新一代引用解析阶段 |

## 图谱与存储

| 主题 | 源码入口 | 读什么 |
|---|---|---|
| 图谱模型 | `gitnexus/src/core/graph/` | KnowledgeGraph 的节点、边、遍历 API |
| LadybugDB schema | `gitnexus/src/core/lbug/schema.ts` | 节点表、关系表、Embedding 表 |
| LadybugDB adapter | `gitnexus/src/core/lbug/lbug-adapter.ts` | init、loadGraphToLbug、CSV 导入、FTS、锁和恢复 |
| CSV 生成 | `gitnexus/src/core/lbug/csv-generator.ts` | 图谱如何流式导出为 CSV |
| FTS | `gitnexus/src/core/search/fts-indexes.ts` | 全文索引创建和校验 |

## MCP 与 Agent

| 主题 | 源码入口 | 读什么 |
|---|---|---|
| MCP server | `gitnexus/src/mcp/server.ts` | tools/resources/prompts 注册、next-step hint |
| Tool definitions | `gitnexus/src/mcp/tools.ts` | `WHEN TO USE`、`AFTER THIS`、input schema |
| LocalBackend | `gitnexus/src/mcp/local/local-backend.ts` | query/context/impact/detect_changes 的真正实现 |
| Resources | `gitnexus/src/mcp/resources.ts` | repo context、schema、processes 等资源 |
| stdout sentinel | `gitnexus/src/mcp/stdio-context.ts` | AsyncLocalStorage 标记 MCP 写入，拦截 stray stdout |
| AI context | `gitnexus/src/cli/ai-context.ts` | AGENTS.md、CLAUDE.md 生成逻辑 |
| Skill generation | `gitnexus/src/cli/skill-gen.ts` | community skill 的生成 |

## Group 跨仓库

| 主题 | 源码入口 | 读什么 |
|---|---|---|
| group CLI | `gitnexus/src/cli/group.ts` | group sync/list/status/impact 命令入口 |
| sync 主流程 | `gitnexus/src/core/group/sync.ts` | 抽取契约、manifest link、exact/wildcard match、writeBridge |
| matching | `gitnexus/src/core/group/matching.ts` | contract normalization、provider index、exact/wildcard |
| bridge DB | `gitnexus/src/core/group/bridge-db.ts` | bridge.lbug 写入 |
| cross impact | `gitnexus/src/core/group/cross-impact.ts` | 跨仓库影响分析扇出 |
| extractors | `gitnexus/src/core/group/extractors/` | HTTP、gRPC、Thrift、Topic、Include、Workspace |

## Web UI

| 主题 | 源码入口 | 读什么 |
|---|---|---|
| Vite React | `gitnexus-web/` | SPA、图谱可视化、聊天面板 |
| HTTP server | `gitnexus/src/server/` | REST、SSE、MCP HTTP endpoint |
| MCP HTTP | `gitnexus/src/server/mcp-http.ts` | Streamable HTTP transport |
