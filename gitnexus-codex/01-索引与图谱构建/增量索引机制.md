# 增量索引机制

增量索引的目标是避免每次 analyze 都做全量重复工作。GitNexus 会结合 commit、文件内容、解析缓存和嵌入缓存来降低重复计算。

## 核心思路

1. 如果 commit 没变且索引可用，可以早期返回。
2. 文件内容没变时，尽量复用解析缓存。
3. Embedding 已存在时，尽量恢复旧向量，避免重复生成。
4. 最终仍以当前仓库状态写出新的 meta 信息。

## 为什么重要

索引阶段包括 AST 解析、图构建、FTS、Embedding 等工作。大型仓库里最贵的是重复解析和重复嵌入，缓存让 analyze 更适合日常使用。

## 边界

增量不是“不重新分析”。当核心配置、语言解析逻辑或依赖图变化时，仍需要全量重建来保证图谱一致。
