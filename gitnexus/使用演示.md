# GitNexus 命令参考

## 一、核心命令（日常最常用）

| 命令 | 用途 |
|------|------|
| gitnexus setup | 一键配置 MCP（对接 Cursor、Claude Code、Codex 等） |
| gitnexus analyze [path] | 索引一个仓库 — 最核心的命令 |
| gitnexus index [path...] | 把已有的 .gitnexus/ 目录注册到全局注册表（不重新分析） |
| gitnexus status | 查看当前仓库的索引状态 |
| gitnexus list | 列出所有已索引的仓库 |
| gitnexus doctor | 诊断运行时平台能力和嵌入配置 |
| gitnexus clean | 删除当前仓库的索引 |
| gitnexus remove <target> | 按别名/名称/路径删除某个已注册仓库的索引 |

## 二、服务器命令

| 命令 | 用途 |
|------|------|
| gitnexus serve | 启动 HTTP 服务器（默认 4747 端口），供 Web UI 连接 |
| gitnexus mcp | 启动 MCP 服务器（stdio），让 AI 编辑器能调用 GitNexus 工具 |
| gitnexus eval-server | 启动轻量 HTTP 评估服务器（默认 4848），用于 SWE-bench |

## 三、直接工具命令（绕过 MCP，用于脚本/CI）

| 命令 | 用途 |
|------|------|
| gitnexus query <搜索词> | 搜索知识图谱中的执行流 |
| gitnexus context [符号名] | 360度查看一个符号：谁调用它、它调用谁 |
| gitnexus impact <目标> | 影响分析：改这个符号会炸到什么 |
| gitnexus cypher <查询> | 直接用 Cypher 查 LadybugDB 图 |
| gitnexus detect-changes | 把 git diff 映射到受影响的符号和流程 |

## 四、Group 命令（多仓库联合分析）

| 命令 | 用途 |
|------|------|
| gitnexus group create <名称> | 创建一个新的仓库组 |
| gitnexus group add <组> <路径> <仓库名> | 把仓库加进组 |
| gitnexus group remove <组> <路径> | 从组里移除仓库 |
| gitnexus group list [名称] | 列出所有组，或查看某个组的详情 |
| gitnexus group status <名称> | 检查组和成员仓库的索引新鲜度 |
| gitnexus group sync <名称> | 同步合约注册表：提取合约并建立跨仓库链接 |
| gitnexus group impact <名称> | 跨仓库影响分析 |
| gitnexus group query <名称> <搜索词> | 跨所有组成员仓库搜索执行流 |
| gitnexus group contracts <名称> | 查看合约注册表内容 |

## 五、其他

| 命令                       | 用途                                          |
| ------------------------ | ------------------------------------------- |
| [[gitnexus wiki]] [path] | 从知识图谱生成仓库 Wiki 文档（需 LLM）                    |
| gitnexus augment <模式>    | 用知识图谱上下文增强搜索模式（hooks 使用）                    |
| gitnexus publish [path]  | 通知 understand-quickly 注册表该仓库有新的 GitNexus 索引 |
